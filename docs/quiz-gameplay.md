# Квиз — геймплей, команды, рейтинг

Бизнес-правила. **Для интеграции фронта см. [quiz-gameplay-api.md](./quiz-gameplay-api.md)** (REST, Socket, payload'ы).

Шаблоны контента: [quiz-templates.md](./quiz-templates.md).

---

## Flow

```text
1. Создать игру → тип quiz → выбрать шаблон (список с q/limit/cursor)
2. Настроить лобби (solo / teams + teamCount) → POST /worlds/:id/games
3. Лобби → join / ready
4. Host стартует → команды (если teams) формируются на сервере
5. Игра: раунды → вопросы (синхронный таймер)
6. Конец раунда — показ изменения очков всем
7. Конец игры — рейтинг мира + персональная сводка (вопрос / ваш ответ / правильный)
```

Настройки solo/teams задаются **при создании сессии** (`quizLobby` в `gameConfig`), **после создания не меняются**.

## Очки за вопрос (игровой счёт сессии)

### Обычный раунд (`isDoublePoints: false`)

| Ситуация | Очки |
|----------|------|
| Первый правильный ответ в вопросе | **+100** |
| Другие правильные (до конца таймера) | **+50** |
| Неправильный / без ответа | **0** |

«Первый» определяет **сервер** по времени получения валидного ответа.

### Раунд с удвоением (`isDoublePoints: true`)

На **каждый** вопрос игрок может включить флаг **«удвоить»** (риск).

| Ситуация | Очки |
|----------|------|
| Правильно + флаг «удвоить» | базовые (100/50) **+ 100** бонус |
| Правильно без флага | базовые (100/50) |
| Неправильно + флаг «удвоить» | **−200** |
| Неправильно без флага | **0** (штрафа нет) |
| Без ответа | **0** |

## Команды

### Лобби

- Host указывает: **solo** или **teams**.
- Если **teams** — число команд `teamCount` (≥ 2).
- **Смены команды в лобби нет** (на будущее).

### Старт игры

- Игроки **случайно** распределяются по командам (разный размер команд **допустим**).
- **Капитан** каждой команды — **случайный** участник команды.

### Ответы в командном режиме

- Все в команде **видят**, кто какой вариант выбрал (черновик / preview).
- **Засчитывается только ответ капитана** (финальная отправка).
- Капитан не ответил до конца таймера → **0** за вопрос (ничего не засчитывается).

## Синхронизация

- Один **серверный таймер** на вопрос (`questionTimeSec` из раунда snapshot).
- Фаза **до раунда**: `preRoundDelaySec` + `description` раунда на экране.
- **Конец раунда**: всем показывается изменение очков (сводка раунда, **8 сек**).
- **Конец игры**: каждому игроку — recap API + socket `quiz:game-finished`.

## Рейтинг в мире (`WorldMember.rating`)

- Поле `WorldMember.rating`, default **0**, **не ниже 0** (clamp).
- Отдаётся в API участников мира.
- Меняется **только по итогам игры**.

### Победа / поражение (не ничья)

| Режим | Победа | Поражение |
|-------|--------|-----------|
| **Solo** | +random **7..10** | −random **7..10** |
| **Teams** | всей команде +random **4..7** | всей команде −random **4..7** |

### Ничья

| Режим | Изменение рейтинга |
|-------|-------------------|
| **Solo** | +random **5..8** (один random на всех при ничьей) |
| **Teams** | всей команде +random **2..5** |

> **Solo-ничья:** один random 5..8 — **одинаковый** бонус всем игрокам с max-счётом.

Победитель solo — max игровых очков; победитель teams — команда с max **суммой** очков команды.

## XP мира (глобальный рейтинг / прогрессия)

| Событие | XP |
|---------|-----|
| Старт игры | **+10** (`GAME_STARTED`) |
| Завершение игры | **+20** (`GAME_FINISHED`) |

Константы: `src/config/world-xp-rewards.ts`.
